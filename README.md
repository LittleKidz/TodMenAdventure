## About The Game

**TodMen Adventure** is a local 2-player tactical battle game built with Java and JavaFX. Two players select unique characters, navigate a randomized 11Г—11 map, collect items, fight Goblins for permanent stat boosts, and battle each other in turn-based combat вҖ” all while a deadly **shrinking lava zone** closes in.

> *Developed as part of the Programming Methodology course (2110215) at the Department of Computer Engineering, Faculty of Engineering, Chulalongkorn University.*

---

##  Key Features

| Feature | Description |
|---|---|
|  **10 Unique Maps** | Randomly selected each game using a shuffle-bag system вҖ” no repeat until all 10 are played |
|  **4 Playable Characters** | Each with unique stats, movement rules, and a special skill |
|  **Shrinking Lava Zone** | Every 8 rounds the safe zone contracts вҖ” stay inside or take damage |
|  **Goblin Encounters** | Defeat Goblins to earn permanent random stat boosts (ATK / DEF / Max HP) |
|  **Item System** | Collect HP Potions, ATK Elixirs, and DEF Elixirs scattered across the map |
|  **Skill Cooldowns** | Each character's special skill has a 3-turn cooldown after use |
|  **Simultaneous Combat** | Both players choose their action secretly, then actions resolve at the same time |

##  Terrain Types

| Tile | Symbol | Description |
|---|:---:|---|
|  **Normal** | `В·` | Open ground вҖ” all characters can pass |
|  **River** | `~` | Impassable вҖ” only **Reborn** can cross |
|  **Tree** | `T` | Impassable вҖ” only **Archer** can pass through |
|  **Rock** | `#` | Blocked for all characters |
|  **Item** | `I` | Step on it to auto-collect a random item |
|  **Goblin** | `G` | Step on it to trigger a battle вҖ” win for a permanent stat boost |
|  **Lava** | `X` | Shrinking zone boundary вҖ” deals **10 HP** damage per round |

##  Battle System

Each battle turn is **simultaneous** вҖ” both players choose an action secretly and they resolve at the same time.

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ            BATTLE ACTIONS               в”Ӯ
в”ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
в”Ӯ  ATTACK    в”Ӯ  Deal ATK вҲ’ DEF damage     в”Ӯ
в”Ӯ  DEFEND    в”Ӯ  Halve all incoming damage в”Ӯ
в”Ӯ  USE ITEM  в”Ӯ  Consume an item from bag  в”Ӯ
в”Ӯ  USE SKILL в”Ӯ  Activate special skill    в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

> **Note:** If your skill is on cooldown or your bag is empty when you choose that action, the game automatically falls back to a normal attack.

##  Installation

### Prerequisites

- **Java 17+** вҖ” [Download here](https://adoptium.net/)
- **JavaFX 17+** вҖ” [Download here](https://openjfx.io/)

### Run the Game

**1. Download the JAR** from [Releases](https://github.com/2110215-ProgMeth/cedt-project-2025-2-todmen/releases) or build from source.

**3. Launch with JavaFX**
```bash
java --module-path {javafx_path} --add-modules
javafx.controls,javafx.fxml,javafx.graphics,javafx.media,javafx.web,javafx.swing 
--enable-nativeaccess=javafx.graphics,javafx.media 
-jar TodMenAdventure-1.ja

```

> рҹ’Ў **Tip:** Replace `/path/to/javafx/lib` with your actual JavaFX SDK `lib` folder path.

##  Architecture

TodMen Adventure follows the **MVC (Model-View-Controller)** pattern:

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ      events       в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ   JavaFX UI  в”Ӯ в—„в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ в”Ӯ    GameManager       в”Ӯ
в”Ӯ  (View)      в”Ӯ                   в”Ӯ    (Model)           в”Ӯ
в”Ӯ              в”Ӯ в”Җв”Җв”Җв”Җ calls в”Җв”Җв”Җв”Җв”Җв”Җв–ә в”Ӯ                      в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ                   в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
       в”Ӯ                                      в”Ӯ
       в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ GameController в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
                    (Controller)
```

**Key Design Patterns:**
- **Observer** вҖ” `GameEventListener` decouples UI from game logic
- **Factory** вҖ” `CharacterType` enum creates player subclasses
- **Strategy** вҖ” `AIControllable` interface for Goblin AI behaviour
- **Template Method** вҖ” `Entity` вҶ’ `Player` вҶ’ concrete characters
